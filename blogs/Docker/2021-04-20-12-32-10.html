# DockerFile

构建步骤：

1、编写一个dockerFile文件

2、docker build构建成为一个镜像

3、docker run 运行镜像

4、docker push 发布镜像（DockerHub、阿里云镜像仓库！）

例如，在docker Hub上查看centos镜像

# 很多官方镜像都是基础包，很多功能没有，我们通常会自己搭建自己的镜像
FROM scratch
ADD centos-8-x86_64.tar.xz /
LABEL \
	org.label-schema.schema-version="1.0"     \
	org.label-schema.name="CentOS Base Image"     \
	org.label-schema.vendor="CentOS"     \
	org.label-schema.license="GPLv2"     \
	org.label-schema.build-date="20200809"
CMD ["/bin/bash"]

# 官方既然可以制作镜像，那我们也可以！

# DockerFile 构建过程

# 基础知识：

1、每个保留关键字（指令）都是必须是大写字母

2、执行从上到下顺序执行

3、# 表示注释

4、每一个指令都会创建提交一个新的镜像层，并提交！

dockerfile是面向开发的，我们以后要发布项目，做镜像，就需要编写dockerfile文件，这个文件十分简单！

Docker镜像逐渐成为企业交付的标准，必须要掌握！

步骤：开发，部署，运维

DockerFile：构建文件，定义了一切的步骤，源代码

DockerImages：通过DockerFile构建生成的镜像，最终发布和运行的产品！

Docker容器：容器就是镜像运行起来提供服务器

# DockerFile的指令

命令 作用
FROM 指定基础镜像
MAINTAINER 指定维护者信息
RUN 编译镜像时运行的脚本
ADD COPY文件，会自动解压
WORKDIR 设置当前工作目录
VOLUME 设置卷，挂载主机目录
EXPOSE 指定对外的端口
CMD 设置容器的启动命令
ENV 设置容器的环境变量
ARG 设置编译镜像时加入的参数
ONBUILD 设置镜像的ONBUILD指令
STOPSIGNAL 设置容器的退出信号量
USER 设置运行RUN CMD ENTRYPOINT 的用户名
LABEL 设置镜像的标签
ENTRYPOINT 设置容器的入口程序

创建一个自己的centos。官网上的centos是阉割版的，很多命令是没有的，然后我要在里面加这些命令，例如：ifconfig，vim等是没有的。

# 1、编写dockerfile
[root@VM_0_10_centos dockerfile]# cat mydockerfile-centos 
FROM centos
MAINTAINER william<chenzhiwei93@aliyun.com>

ENV MYPATH /usr/local
WORKDIR $MYPATH
RUN yum -y install vim
RUN yum -y install net-tools

EXPOSE 80

CMD echo $MYPATH
CMD echo "----end----"
CMD /bin/bash

# 2、通过这个文件构建镜像（注意命令后面有个.）
# 命令 docker build -f dockerfile文件路径 -t 镜像名:[tag] .
[root@VM_0_10_centos dockerfile]# docker build -f mydockerfile-centos -t mycentos:0.1 .
